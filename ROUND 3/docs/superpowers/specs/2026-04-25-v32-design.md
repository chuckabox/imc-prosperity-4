# trader_ken_v32 — Design Spec
Date: 2026-04-25

## Problem

`trader_ken_v30_standalone.py` earns ~4–5k XIRECs vs competitors at ~100k. Root causes:
1. Static `HP_ANCHOR=9993` creates opening-tick drawdowns when live price differs.
2. No order-book imbalance signal — the primary alpha source in HYDROGEL is ignored.
3. VEV module tracks observed prices (not BS fair) — no directional alpha extracted.
4. Global risk governor throttles all modules simultaneously, compounding caution.

## Architecture

Single file `trader_ken_v32.py`. Three independent modules, no cross-module coupling.

```
run(state)
 ├── _load_state()
 ├── _hp_logic(state)           → List[Order]
 ├── _vfe_logic(state)          → List[Order]
 ├── _vev_logic(state, vfe_mid) → List[Order]
 └── _save_state()
```

State (traderData JSON):
```json
{
  "hp_ewma": null,
  "vfe_ewma": null
}
```

## Module 1: HYDROGEL_PACK

**Position limit:** 80

**Fair value:**
- EWMA of mid, alpha=0.003 (half-life ≈ 300 ticks, matching measured OU).
- Initialize to first observed mid. No static anchor.

**Imbalance signal:**
```
imb = (bid_vol_1 - ask_vol_1) / (bid_vol_1 + ask_vol_1)
```
- `imb == 0` (97% of ticks): quote symmetrically at `fair ± EDGE` (EDGE=8).
- `imb > 0` (price walking up): shift both quotes up by SKEW=4.
  - bid = fair - EDGE + SKEW, ask = fair + EDGE + SKEW
- `imb < 0` (price walking down): shift both quotes down by SKEW=4.
  - bid = fair - EDGE - SKEW, ask = fair + EDGE - SKEW
- If `imb != 0` and current position is on the wrong side of the move: fire a taker (up to 5 lots) to reduce exposure.

**Inventory skew:** If `|pos| > 40`: add `pos * 0.15` to both quote prices (lean against position passively).

**Sizing:** Quote full room-to-limit on both sides. No hard maker_max cap.

## Module 2: VELVETFRUIT_EXTRACT

**Position limit:** 60

**Fair value:** EWMA of mid, alpha=0.05.

**Quoting:** Passive maker at `fair ± 3`. Full room-to-limit size.

**Inventory skew:** If `|pos| > 30`: add `pos * 0.1` to quotes.

No taker logic. No trend chasing.

## Module 3: VEV Options (5200 + 5300 only)

**Position limit:** 20 per strike.

**BS fair value:**
- sigma = 0.0176 per day (bias-corrected realized vol from data analysis).
- TTE = `(8 - day) - timestamp / 1_000_000` (exact, not just 8-day).
- `norm_cdf` implemented inline via Abramowitz & Stegun rational approximation.
- Standard Black-Scholes call: `C = S*N(d1) - K*exp(-r*TTE)*N(d2)`, r=0.
- Computed from `S = mid(VELVETFRUIT_EXTRACT)`.

**Entry logic:**
- If `market_ask < bs_fair - 5`: buy up to VEV_LIMIT (taker).
- If `market_bid > bs_fair + 5`: sell up to VEV_LIMIT (defensive taker).
- No market-making. No delta hedging.

**Strikes traded:** VEV_5200 and VEV_5300 only. All other strikes ignored.

## Risk Management

Per-module only: each module caps orders at `limit - |pos|`. No global risk score, no speed limiters, no open-phase throttle.

## Parameters Summary

| Parameter | Value | Rationale |
|---|---|---|
| HP_LIMIT | 80 | Confirmed working in v30/live |
| HP_EWMA_ALPHA | 0.003 | Matches OU half-life ~300 ticks |
| HP_EDGE | 8 | Half of typical 16-wide spread |
| HP_SKEW | 4 | Matches ~4 XIREC imbalance-predicted move |
| HP_INV_TRIGGER | 40 | Half of limit |
| HP_INV_FACTOR | 0.15 | Gentle lean |
| HP_TAKER_MAX | 5 | Small defensive unwind only |
| VFE_LIMIT | 60 | Reduced from v30's 80 for safety |
| VFE_EWMA_ALPHA | 0.05 | Tracks VFE's faster diffusion |
| VFE_EDGE | 3 | Tight passive spread |
| VFE_INV_TRIGGER | 30 | Half of limit |
| VFE_INV_FACTOR | 0.10 | Gentle lean |
| VEV_LIMIT | 20 | Conservative, minimal voucher alpha |
| VEV_SIGMA | 0.0176 | Bias-corrected realized vol |
| VEV_MIN_EDGE | 5 | Minimum XIRECs edge to enter |
| VEV_STRIKES | [5200, 5300] | Highest gamma, best edge/spread ratio |

## What is NOT in v32

- No VEV_5000, VEV_5100, VEV_5400, VEV_5500 (too small edge or too deep ITM/OTM).
- No delta hedging between VEV and VFE (adds position conflicts, complex).
- No global risk governor (throttled all modules simultaneously in v30).
- No speed limiters or open-phase throttle (caused the day-start drawdown).
- No static anchors (source of mean drift errors).
